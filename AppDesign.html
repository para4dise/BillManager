1. 앱 개요

1.1 앱 소개

Bill Manager는 신용카드, 통신비, 렌트 등 정기적으로 발생하는 청구서의 연체를 방지하기 위한 모바일 애플리케이션입니다. 사용자가 납부 기한을 놓치지 않도록 직관적인 인터페이스와 스마트 알림 기능을 제공하며, 모든 데이터는 로컬에 저장되어 개인정보 보호를 보장합니다.

핵심 목표

  • 청구서 납부 기한 관리 자동화
  • 연체 방지를 위한 시각적 경고 시스템
  • 간편한 납부 상태 추적
  • 다국어 및 다중 화폐 지원

1.2 기술 스택

Frontend

React Native Expo React Navigation i18next

Backend

SQLite Expo FileSystem AsyncStorage

Feature

CSV Import/Export Local Notifications Date-fns Gesture Handler

1.3 주요 화면 구성

Tab 1: Payment Dashboard

  • 향후 3개월간의 납부 일정 타임라인 뷰
  • 납부 상태별 시각적 구분
    • Overdue (빨간색)
    • Due Soon (분홍색)
    • Paid (회녹색)
    • Unpaid (회색)
  • 월별 납부 현황 요약 (Paid / Total)
  • 빠른 상태 변경 (탭, 스와이프)

Tab 2: Account Management

  • 등록된 계정 목록 (최신순)
  • 계정별 상세 정보 관리
  • 반복 청구 설정
  • 납부 이력 조회

Tab 3: Settings

  • 프로필 설정 (이름, 언어, 화폐)
  • 알림 설정
  • 데이터 백업/복원 (CSV Import/Export)
  • 개발자 모드 (로그 뷰어, 데이터 재생성, 초기화)

2. 구조

2.1 데이터 모델 (DB Tables & Fields)

Table: accounts

id INTEGER PRIMARY KEY AUTOINCREMENT name TEXT NOT NULL category TEXT NOT NULL (Credit Card, Rent, Utility, etc.) bank_account TEXT notes TEXT amount REAL currency TEXT DEFAULT 'USD' repeats TEXT (Monthly, Weekly, Yearly, etc.) start_date TEXT (ISO 8601) has_end_date INTEGER (0 or 1) end_date TEXT (ISO 8601) is_active INTEGER DEFAULT 1 created_at TEXT (ISO 8601 UTC) updated_at TEXT (ISO 8601 UTC)

Table: payments

id INTEGER PRIMARY KEY AUTOINCREMENT account_id INTEGER (FK to accounts) due_date TEXT (ISO 8601, date only) amount REAL is_paid INTEGER DEFAULT 0 (0: Unpaid, 1: Paid) paid_date TEXT (ISO 8601) note TEXT created_at TEXT (ISO 8601 UTC) updated_at TEXT (ISO 8601 UTC)

Table: reminders

id INTEGER PRIMARY KEY AUTOINCREMENT account_id INTEGER (FK to accounts) remind_value INTEGER (e.g., 3) remind_unit TEXT (day, week, month) remind_type TEXT (due, before_due) is_active INTEGER DEFAULT 1 created_at TEXT (ISO 8601 UTC)

Table: payment_methods

id INTEGER PRIMARY KEY AUTOINCREMENT name TEXT NOT NULL type TEXT (Bank Transfer, Credit Card, Cash, etc.) is_active INTEGER DEFAULT 1 created_at TEXT (ISO 8601 UTC)

Table: logs

id INTEGER PRIMARY KEY AUTOINCREMENT action TEXT NOT NULL table_name TEXT record_id INTEGER details TEXT (JSON format) timestamp TEXT (ISO 8601 UTC)

Table: settings

id INTEGER PRIMARY KEY AUTOINCREMENT key TEXT UNIQUE NOT NULL value TEXT updated_at TEXT (ISO 8601 UTC)

2.3 자동화 로직

납부 일정 자동 생성 (Payment Auto-generation)

  • Account의 repeats 설정에 따라 향후 3개월치 Payment 레코드 자동 생성
  • 앱 실행 시 또는 Account 생성/수정 시 트리거
  • 중복 생성 방지: 동일 account_id와 due_date 조합 체크
  • end_date 설정 시 해당 날짜까지만 생성

알림 스케줄링

  • Reminders 테이블 기반으로 로컬 알림 예약
  • 매일 자정에 다음 날 알림 재계산
  • 알림 시간은 Settings의 notification_time 기준

상태 기반 색상 분류

상태 조건 색상
Overdue 현재 날짜 > due_date && is_paid = 0 빨간색
Due Soon due_date - 현재 날짜 ≤ 3일 && is_paid = 0 분홍색
Paid is_paid = 1 회녹색
Unpaid (일반) 그 외 회색

3. 주요 기능

3.1 자동 결제 일정 생성

  • 반복 주기 설정: Weekly, Monthly, Yearly, Custom
  • 시작일 지정: Calendar Picker로 첫 납부일 선택
  • 종료 조건: 무기한 또는 특정 날짜까지
  • 자동 생성 로직:
    • Account 저장 시 향후 3개월치 Payment 레코드 생성
    • 기존 레코드 중복 체크 (account_id + due_date)
    • 월말 처리: 31일을 기준으로 하는 경우 각 월의 마지막 날로 조정

3.2 스마트 알림

  • 다중 알림 설정: 계정당 여러 개의 알림 등록 가능
  • 알림 타이밍:
    • 납부 당일
    • N일/주/월 전
  • 알림 시간 통합: 모든 알림은 설정된 하나의 시간에 발송
  • 알림 내용: 계정명, 금액, 납부 기한 포함

3.3 상태 관리

  • 빠른 상태 변경:
    • 탭: Payment Detail 화면으로 이동
    • 좌 스와이프: Paid로 변경
    • 우 스와이프: Unpaid로 변경
  • 자동 타임스탬프: 상태 변경 시 paid_date 자동 기록
  • 메모 기능: 각 납부 건에 노트 추가 가능
  • 납부 이력: Account별 과거 납부 기록 조회 (최신순)

3.4 다국어 지원

  • 지원 언어: 영어(en), 한국어(ko), 일본어(ja) - 확장 가능
  • 번역 범위:
    • UI 레이블 및 버튼
    • 카테고리명
    • 오류 메시지
    • 날짜 형식 로컬라이제이션
  • 구현: i18next 라이브러리 활용
  • 기본 언어: 디바이스 언어 자동 감지, 미지원 시 영어

4. UI/UX

4.1 과하지 않은 간결한 인터페이스

컬러 스킴

  • 기본: 은은한 회색과 검정색 (미니멀)
  • Overdue: 빨간색 (#FF4444)
  • Due Soon: 분홍색 (#FFB6C1)
  • Paid: 회녹색 (#95A99B, 낮은 opacity)

타이포그래피

  • 계층 구조 명확 (계정명 > 금액 > 날짜)
  • 가독성 우선 (충분한 행간, 적절한 폰트 크기)

레이아웃

  • 카드 형식으로 정보 그룹화
  • 여백을 통한 시각적 구분

4.2 빠른 액션 버튼과 Swipe 동시 사용

  • 스와이프 제스처:
    • 좌: Paid 마크 (체크 아이콘)
    • 우: Unpaid 마크 (X 아이콘)
  • 탭 동작: Detail 화면 전환
  • 롱프레스: 추가 옵션 메뉴 (삭제, 수정)

4.3 원터치 상태 변경

Payment Detail 화면:
  • Toggle Switch로 Paid/Unpaid 즉시 변경
  • 변경 시 실시간 저장
  • 변경 내역 로그 자동 기록

4.4 메모 기능

용도:

  • 납부 방법 기록 (계좌이체, 카드 등)
  • 특이사항 메모
  • 납부 확인 번호

UI: Multi-line TextInput

저장: payments 테이블의 note 필드

4.5 데이터 백업과 복원

Export (백업)

  • 형식: CSV (UTF-8 with BOM)
  • 내용:
    • accounts.csv: 모든 계정 정보
    • payments.csv: 모든 납부 기록
    • reminders.csv: 알림 설정
  • 위치: 디바이스 Documents 폴더 또는 공유 기능 제공
  • 타임스탬프: 파일명에 날짜 포함 (BillManager_backup_20250929.zip)

Import (복원)

  • 파일 선택: Document Picker
  • 유효성 검증:
    • CSV 헤더 확인
    • 데이터 타입 체크
    • 중복 키 처리
  • 오류 처리:
    • 실패 시 롤백
    • 오류 로그 제공

4.6 모든 작업 로그 처리 및 뷰어

로그 기록 대상

  • Account 생성/수정/삭제
  • Payment 상태 변경
  • 데이터 Import/Export
  • 자동 Payment 생성
  • 설정 변경

로그 뷰어 기능

  • 필터링:
    • 날짜 범위
    • 작업 유형 (action)
    • 테이블명
  • 정렬: 최신순 기본, 오름차순/내림차순 전환
  • 상세 보기: JSON 형식의 details 내용 표시
  • 검색: 키워드 검색 기능
  • 내보내기: 로그를 텍스트 파일로 저장

4.7 전체 데이터베이스 테이블 초기화 기능

개발자 모드 메뉴

  • Export Data: 현재 모든 데이터 CSV 백업
  • Import Data: CSV 파일에서 데이터 복원
  • Regenerate Payments (Skip Existing): 기존 Payment 유지하며 누락된 건만 생성
  • Regenerate Payments (Delete & Recreate): 모든 Payment 삭제 후 재생성 (3개월치)
  • Data Reset All: 모든 테이블 데이터 삭제 (DROP 후 CREATE)
